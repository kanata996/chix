# 测试口径

## 1. 目标

测试应优先验证对外可观察的 HTTP 行为，而不是证明内部经过了多少命名阶段。

## 2. 黑盒优先

推荐把测试重点放在：

- 状态码
- 响应 body 结构
- 关键 header
- 写回后不可改写

## 3. 最小测试矩阵

至少应覆盖：

- 非法 JSON -> `400`
- 请求字段错误 -> `422`
- 未认证或准入拒绝 -> 对应 `401/403`
- route 不存在 -> `404`
- method 不允许 -> `405`
- rate limit -> `429`
- 业务资源不存在 -> `404`
- 业务冲突 -> `409`
- 业务规则不成立 -> `422`
- 内部 panic 或写回前内部失败 -> `500`
- 成功响应 -> 正确的 success envelope
- 响应开始后出错 -> 不改写已有响应

## 4. 不再强调的测试

当前有效规范不再要求围绕以下内容建立大规模测试矩阵：

- 旧边界理论里的提前终止独立分类
- “双语义线”
- 旧阶段切换点术语
- 固定 phase/cut-point 术语

## 5. 测试风格建议

- 优先使用端到端或 handler 黑盒测试
- 只有在映射逻辑复杂时，才额外补 mapper 单测
- 测试名称应描述输入和输出，而不是内部理论阶段
